How to read the result

  • VALID means the server can verify the attestation and the device is in a normal state.
  • FORKED or REVOKED means the attestation exists, but the device identity is no longer in a normal state.
  • UNKNOWN means the verification ticket was not found.
  • Key rotation events show how many times the device identity has been rotated on the server.
