# Canvas Design

## Overview

Turn a brief into an original visual philosophy and then express it as a refined static composition. Prioritize design quality, spatial control, and subtle conceptual depth over literal illustration or heavy explanation.

## Workflow

1. Distill the request into a subtle conceptual thread rather than a literal prompt.
2. Write a named design philosophy as a `.md` artifact before making the canvas.
3. Use that philosophy to produce a single polished `.png` or `.pdf` unless the user asks for multiple pages.
4. Refine the first composition before adding new elements; polish matters more than quantity.

## Output Requirements

- Save the philosophy as a `.md` file.
- Save the final visual artifact as a `.png` or `.pdf`.
- Keep text sparse and design-forward.
- Make every element feel deliberate, aligned, and contained within the canvas.

## Working Style

- Favor abstract or editorial sophistication over cartoony or obvious interpretations.
- Use typography as part of the composition, not an afterthought.
- Treat the source subject as a quiet conceptual reference embedded into form, rhythm, color, and structure.
- Reach for highly crafted restraint before decorative excess.
