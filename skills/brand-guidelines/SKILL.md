# Brand Guidelines

## Overview

Restyle the artifact without changing its underlying message. Preserve structure and improve visual consistency so the result feels intentionally branded rather than merely recolored.

## Workflow

1. Inspect the artifact type and the amount of styling freedom available.
2. Preserve content, information hierarchy, and legibility before making visual changes.
3. Apply the brand palette and typography consistently across headings, body text, code, shapes, and accents.
4. Check contrast, spacing, and fallback fonts before finishing.

## Brand System

### Colors

- Charcoal: `#0E0F12` for primary text and dark backgrounds
- Slate: `#202123` for surface backgrounds
- Mist: `#F5F7FA` for light backgrounds and text on dark fields
- Steel: `#9EA1AA` for secondary text, dividers, and quieter UI elements
- OpenAI Green: `#10A37F` for the primary accent
- Azure: `#2B8FFF` for secondary emphasis
- Graphite: `#40434A` for neutral strokes, icons, and restrained accents

### Typography

- Use `Inter` for headings and body text when available.
- Use semibold or bold weights for headings and regular for body text.
- Use `IBM Plex Mono` for code or data-forward monospace treatments.
- Fall back to `Arial` for sans-serif text and `Menlo` or `monospace` for code when custom fonts are unavailable.

## Application Rules

- Favor Charcoal, Slate, and Mist as the base system; use accent colors deliberately, not everywhere.
- Keep heading styles visually distinct from body text through weight, scale, and spacing.
- Style code, numbers, or technical labels with monospace only when that improves clarity.
- Reuse one or two accent colors per artifact unless the artifact already needs richer categorization.
- Preserve readability on both light and dark backgrounds by switching text to Charcoal or Mist as needed.
- Avoid decorative effects that fight the brand system, including low-contrast gradients, noisy shadows, or random color variation.

## Output Standard

- Keep the artifact recognizable, only more polished and on-brand.
- Make changes consistently across all sections, not just the title area.
- If the source artifact already has strong structure, refine it instead of rebuilding it from scratch.
